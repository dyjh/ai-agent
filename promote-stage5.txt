任务 3：正式实现 interrupt-resume
==========================

这是本阶段核心。

要求：

1. 当 PolicyDecision 判定 requires_approval=true：

   * 生成 approval request
   * 保存 approval.input_snapshot
   * run 进入 paused_for_approval
   * 发送 approval.requested 事件
   * Start 返回的 event stream 正常结束或进入 paused 终点

2. 当用户 approve：

   * Workflow.Resume(runID, approvalID, true)
   * 读取 run state
   * 读取 approval
   * 确认 approval 属于该 run
   * 状态迁移到 approval_approved
   * 直接执行 approval.input_snapshot
   * 不能重新调用 Planner 生成新 proposal
   * 执行完继续总结并 completed

3. 当用户 reject：

   * Workflow.Resume(runID, approvalID, false)
   * 状态迁移到 approval_rejected
   * 不执行 Executor
   * 生成一条 assistant summary，例如说明“该操作未执行，因为审批被拒绝”
   * run 进入 completed 或 cancelled（你选择一种，但要一致）

4. Resume 必须可重复防御：

   * 已审批过的 approval 不应重复执行
   * 错误 approvalID 应报错
   * approval/run 不匹配应报错

建议增加：

```go
type PersistedRunStateStore interface {
    Save(ctx context.Context, state *RunState) error
    Get(ctx context.Context, runID string) (*RunState, error)
    Update(ctx context.Context, state *RunState) error
}
```

如果当前项目已有 JSONL/agent_events/DB 结构，优先复用，不要重复造太多存储层。
