## 7.3 实现真实 QdrantVectorIndex

实现 interface：

```go
type VectorChunk struct {
    ID         string
    Text       string
    Vector     []float32
    Payload    map[string]any
    SourceFile string
}

type VectorSearchResult struct {
    ID      string
    Text    string
    Score   float32
    Payload map[string]any
}

type VectorIndex interface {
    EnsureCollections(ctx context.Context) error
    UpsertChunks(ctx context.Context, collection string, chunks []VectorChunk) error
    DeleteBySourceFile(ctx context.Context, collection string, sourceFile string) error
    Search(ctx context.Context, collection string, query string, filters map[string]any, topK int) ([]VectorSearchResult, error)
    Health(ctx context.Context) error
}
```

QdrantVectorIndex 要求：

* 使用 Go Qdrant client 或稳定 HTTP client 封装。
* 支持 EnsureCollections。
* 支持 collection 不存在时创建。
* 支持 upsert。
* 支持 delete by source_file。
* 支持 search。
* 支持 payload filter，至少支持：

  * kb_id
  * document_id
  * source_file
  * memory_type
  * scope
  * project_key
* 支持 Health。
* 支持 timeout。
* 支持 API key header。
* Qdrant payload 不要存敏感内容。
* Payload 可以存 source_file、chunk_id、kb_id、document_id、section、content_hash、updated_at，但不要把完整长期记忆或密钥类内容放 payload。
* Text 是否作为 payload 存储要谨慎：KB chunk text 可用于搜索返回，但 Memory 长期事实源仍是 Markdown。可以把 chunk text 作为 point payload 的 `text`，但 README 要说明 Qdrant 仍只是索引，Markdown/KB 文件才是事实源。
